Health Vault — Implementation Detail
Health entries — 5 entry limit logic
// Query for free users — show only 5 most recent across ALL types SELECT * FROM health_entries WHERE dog_id = :dogId ORDER BY date DESC LIMIT 5; -- free users only // Query for Plus/Pro — full history SELECT * FROM health_entries WHERE dog_id = :dogId ORDER BY date DESC; -- no limit
⚠️
Never delete free user data
All entries are saved to the DB regardless of tier. Free users see last 5 but all entries exist. When they upgrade to Plus, full history appears immediately — no data migration needed.
Vaccinations & Medications — always free

Vaccinations and medications are never gated. These are safety-critical. All users see all records regardless of tier. No limit applied.

Health documents — Plus/Pro only
// Before allowing file upload, check tier if (!profile.isPlus) { showPaywallModal(feature: 'health_documents'); return; } // Proceed with Supabase Storage upload
Vet report export — Plus/Pro only
  • Button always visible to all users — greyed out with lock icon for Free
  • Tapping triggers paywall modal for Free users
  • PDF generated client-side from Health Vault data — no server call needed
  • Export via iOS share sheet (AirDrop, email, WhatsApp, save to Files)

Ask Sunday — Session Gate
What counts as one session

One session = one conversation thread. Starting a new chat = consuming one session. Continuing an existing thread = free, no additional session consumed.

🚨
Sessions reset on the 1st of each month
Use the month_year field in ask_sunday_sessions table (format: YYYY-MM). Count rows where user_id matches and month_year = current month. Reset is automatic — new month, new count.
Session limit by tier
TierThreads / monthBehaviour at limit
Free1Show paywall modal on new thread attempt
Plus3Show paywall modal — upgrade to Pro for unlimited
ProUnlimitedNo gate
Session check logic (Dart)
// Before starting a new Ask Sunday thread final monthYear = DateFormat('yyyy-MM').format(DateTime.now()); final count = await supabase .from('ask_sunday_sessions') .select() .eq('user_id', userId) .eq('month_year', monthYear) .count(); final limit = profile.isPro ? 999 : profile.isPlus ? 3 : 1; if (count >= limit) { showPaywallModal(feature: 'ask_sunday'); return; } // Create new session row, then open chat await supabase.from('ask_sunday_sessions').insert({ 'user_id': userId, 'dog_id': dogId, 'month_year': monthYear, });

Modal structure
  • Top: Mascot image (mascot_happy.png) — friendly, not punishing
  • Headline: Feature-specific (see copy below)
  • Subtext: What they unlock and at what tier
  • Primary CTA: "Upgrade to Plus — S$8.99/mo" or "Upgrade to Pro — S$15.99/mo"
  • Secondary CTA: "See all plans" → full pricing screen
  • Dismiss: Small "Not now" text link at bottom — always allow dismissal

Never block the user
The paywall modal must always be dismissable. Apple App Store guidelines require that free features remain accessible. Never hard-block navigation behind a paywall.

Subscription Lapse Handling
What happens when a subscription lapses
  • RevenueCat webhook updates is_plus / is_pro to false in profiles table
  • User is immediately downgraded to Free tier on next app open
  • All data is preserved — health entries, spending, documents all remain in DB
  • User sees only last 5 health entries and last 5 spending entries again
  • If they re-subscribe, full history returns instantly
💡
Grace period
RevenueCat handles a 16-day grace period for billing failures before marking as lapsed. Do not implement custom grace period logic — trust RevenueCat's entitlement status.
Lapse notification (in-app)

On first app open after lapse, show a single non-blocking banner (not a modal):

"Your Sunday Plus subscription has ended. Your data is safe — resubscribe anytime to access it."

Banner dismissable. Links to pricing screen. Do not show again for 7 days.

Restore purchases

Add a "Restore Purchases" button in Profile → Support the App. Required by Apple App Store guidelines. Calls RevenueCat's restorePurchases() method.
